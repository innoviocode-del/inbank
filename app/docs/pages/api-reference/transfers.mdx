# Transfers

Transfers move funds between accounts, wallets, and external bank accounts. Supports multiple payment rails including ACH, Wire, SEPA, SWIFT, PIX, and SPEI across fiat and crypto.

## The Transfer Object

```json
{
  "id": "txfr_abc123",
  "object": "transfer",
  "client_reference_id": "your_reference_123",
  "amount": "10000.00",
  "currency": "usd",
  "on_behalf_of": "cus_xyz789",
  "developer_fee": "10.00",
  "source": {
    "payment_rail": "ach_push",
    "currency": "usd"
  },
  "destination": {
    "payment_rail": "ethereum",
    "currency": "usdc",
    "to_address": "0x71C7656EC7ab88b098defB751B7401B5f6d8976F"
  },
  "state": "payment_processed",
  "receipt": {
    "initial_amount": "10000.00",
    "developer_fee": "10.00",
    "exchange_fee": "5.00",
    "subtotal_amount": "9985.00",
    "gas_fee": "0.50",
    "final_amount": "9984.50"
  },
  "created_at": "2026-01-15T10:00:00.000Z",
  "updated_at": "2026-01-15T10:00:02.000Z"
}
```

### Attributes

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique identifier (UUID) |
| `client_reference_id` | string | Your reference ID (max 256 chars) |
| `amount` | string | Transfer amount as decimal string |
| `currency` | string | Fiat currency (usd, eur, mxn, brl, gbp) |
| `on_behalf_of` | string | Customer ID for the transfer |
| `developer_fee` | string | Your fee deducted from transfer |
| `developer_fee_percent` | string | Your fee as percentage (0.0-100.0) |
| `source` | object | Source account/wallet/bank |
| `destination` | object | Destination account/wallet/bank |
| `state` | string | Transfer state (see lifecycle below) |
| `source_deposit_instructions` | object | Instructions for customer deposits |
| `receipt` | object | Fee breakdown and final amounts |
| `return_details` | object | Return information if applicable |
| `created_at` | string | ISO8601 creation timestamp |
| `updated_at` | string | ISO8601 last update timestamp |

## Create a Transfer

```
POST /v0/transfers
```

### Headers

| Header | Required | Description |
|--------|----------|-------------|
| `Api-Key` | Yes | Your API key |
| `Idempotency-Key` | Yes | Unique key for idempotent requests |

### Request Body

```json
{
  "amount": "1000.00",
  "on_behalf_of": "cus_xyz789",
  "developer_fee": "5.00",
  "client_reference_id": "your_ref_123",
  "source": {
    "payment_rail": "ethereum",
    "currency": "usdc",
    "from_address": "0x71C7656EC7ab88b098defB751B7401B5f6d8976F"
  },
  "destination": {
    "payment_rail": "ach",
    "currency": "usd",
    "external_account_id": "ext_abc123"
  }
}
```

### Request Parameters

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `amount` | string | Yes* | Transfer amount (omit if `flexible_amount` enabled) |
| `on_behalf_of` | string | Yes | Customer ID |
| `developer_fee` | string | No | Your fixed fee amount |
| `developer_fee_percent` | string | No | Your fee as percentage |
| `client_reference_id` | string | No | Your reference ID |
| `dry_run` | boolean | No | Validate without creating |
| `source` | object | Yes | Source configuration |
| `destination` | object | Yes | Destination configuration |
| `features` | object | No | Feature flags |

### Features

```json
{
  "features": {
    "flexible_amount": true,
    "static_template": true,
    "allow_any_from_address": true
  }
}
```

| Feature | Description |
|---------|-------------|
| `flexible_amount` | Accept variable amounts (omit `amount` from request) |
| `static_template` | Use static deposit instructions |
| `allow_any_from_address` | Accept deposits from any address |

## Transfer State Lifecycle

```
awaiting_funds → funds_received → in_review → payment_submitted → payment_processed
                                           ↘ canceled
                                           ↘ undeliverable → returned / refunded
```

| State | Description |
|-------|-------------|
| `awaiting_funds` | Waiting for customer deposit |
| `funds_received` | Deposit confirmed, processing |
| `in_review` | Under compliance review |
| `payment_submitted` | Payment sent to destination |
| `payment_processed` | Transfer complete |
| `canceled` | Transfer canceled |
| `undeliverable` | Cannot deliver to destination |
| `returned` | Funds returned to source |
| `refunded` | Refund issued to customer |

## Receipt Object

The receipt provides a breakdown of fees and final amounts:

```json
{
  "receipt": {
    "initial_amount": "1000.00",
    "developer_fee": "5.00",
    "exchange_fee": "2.50",
    "subtotal_amount": "992.50",
    "gas_fee": "0.50",
    "final_amount": "992.00",
    "remaining_prefunded_balance": "50000.00",
    "source_tx_hash": "0xabc123...",
    "destination_tx_hash": "0xdef456...",
    "exchange_rate": "0.91",
    "url": "https://dashboard.lux.financial/transaction/.../receipt/..."
  }
}
```

| Field | Description |
|-------|-------------|
| `initial_amount` | Original transfer amount |
| `developer_fee` | Your platform fee |
| `exchange_fee` | Lux processing fee |
| `subtotal_amount` | Amount after fixed fees |
| `gas_fee` | Blockchain gas costs (if applicable) |
| `final_amount` | Amount received by destination |
| `exchange_rate` | Currency conversion rate (if applicable) |
| `source_tx_hash` | Source blockchain transaction |
| `destination_tx_hash` | Destination blockchain transaction |
| `url` | Customer-facing receipt URL |

## SDK Example

```typescript
const transfer = await lux.transfers.create({
  amount: '1000.00',
  on_behalf_of: 'cus_xyz789',
  developer_fee: '5.00',
  source: {
    payment_rail: 'ethereum',
    currency: 'usdc',
    from_address: '0x71C7656EC7ab88b098defB751B7401B5f6d8976F',
  },
  destination: {
    payment_rail: 'ach',
    currency: 'usd',
    external_account_id: 'ext_abc123',
  },
}, {
  idempotencyKey: 'unique-key-123',
});

// Poll for completion
const completed = await lux.transfers.retrieve(transfer.id);
console.log(completed.state); // "payment_processed"
console.log(completed.receipt.final_amount); // "992.00"
```
